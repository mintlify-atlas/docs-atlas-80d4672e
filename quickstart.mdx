---
title: "Get started with HireFlow"
description: "Clone the repo, configure your environment variables, and run the backend, Celery worker, and Next.js frontend locally in under ten minutes."
---

This guide walks you through running the full HireFlow stack on your machine. You will start the FastAPI backend on port 8000, a Celery worker for async AI tasks, and the Next.js frontend on port 3000.

## Prerequisites

- Python 3.10+
- Node.js 18+
- Redis running locally (default: `redis://localhost:6379`)
- An OpenAI API key

<Steps>

<Step title="Clone the repository and configure environment files">

Clone the project and copy the example environment files:

```bash
git clone https://github.com/akshatbh00/GenAI-Easy-Hire
cd instant_hire

# Copy backend env template
cp backend/.env.example backend/.env

# Set the frontend API URL
echo "NEXT_PUBLIC_API_URL=http://localhost:8000/api/v1" > frontend/.env.local
```

Open `backend/.env` and set at minimum these three values:

| Variable | Description |
|---|---|
| `OPENAI_API_KEY` | Your OpenAI API key — used for GPT-4o-mini and embeddings |
| `JWT_SECRET` | A long random string used to sign authentication tokens |
| `DATABASE_URL` | SQLite path for dev, e.g. `sqlite:///./hireflow.db` |

<Tip>
  All other variables (Redis URLs, storage backend, vector backend) have sensible defaults for local development. You only need the three above to get started.
</Tip>

</Step>

<Step title="Set up the Python backend">

Create a virtual environment and install dependencies:

```bash
cd backend
python -m venv venv
```

<CodeGroup>

```bash Windows
venv\Scripts\activate
```

```bash Mac / Linux
source venv/bin/activate
```

</CodeGroup>

Install all required packages:

```bash
pip install fastapi uvicorn sqlalchemy pydantic-settings \
            pydantic[email] python-jose[cryptography] \
            passlib[bcrypt] python-multipart celery redis \
            openai pymupdf python-docx loguru boto3 \
            pgvector alembic sendgrid numpy
```

Run the database migration to create all tables:

```bash
python -c "from database import Base, engine; from models import *; Base.metadata.create_all(engine)"
```

Start the API server:

```bash
uvicorn main:app --reload --port 8000
```

The API is now running at `http://localhost:8000`. The interactive docs are at `http://localhost:8000/api/v1/docs`.

</Step>

<Step title="Start the Celery worker">

Open a new terminal window, navigate back to the `backend` directory, activate the same virtual environment, then start the worker:

```bash
cd instant_hire/backend
```

<CodeGroup>

```bash Windows
venv\Scripts\activate
celery -A workers.celery_app worker --loglevel=info
```

```bash Mac / Linux
source venv/bin/activate
celery -A workers.celery_app worker --loglevel=info
```

</CodeGroup>

<Warning>
  The Celery worker must be running for resume uploads to be processed. Without it, AI analysis, ATS scoring, and embedding generation will not complete.
</Warning>

</Step>

<Step title="Start the Next.js frontend">

In a third terminal window, install dependencies and start the dev server:

```bash
cd instant_hire/frontend
npm install
npm run dev
```

The frontend is now available at `http://localhost:3000`.

</Step>

</Steps>

## Key URLs

Once all three processes are running, use these URLs:

| Service | URL |
|---|---|
| Frontend | `http://localhost:3000` |
| Job seeker dashboard | `http://localhost:3000/dashboard` |
| Jobs board | `http://localhost:3000/jobs` |
| Recruiter dashboard | `http://localhost:3000/company/dashboard` |
| API interactive docs | `http://localhost:8000/api/v1/docs` |
